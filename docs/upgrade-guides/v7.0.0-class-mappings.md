# v7.0.0 Class Mappings Reference

**Version**: 7.0.0  
**Created**: 2026-02-09  
**Purpose**: Reference table for all deprecated class mappings in v7.0.0

## Overview

This document provides a comprehensive mapping of all classes that were moved, renamed, or namespaced in Application Framework v7.0.0. All deprecated classes maintain backward compatibility temporarily through wrapper classes in `src/classes/_deprecated/`.

## How to Use This Document

- **Old Class**: The class name used in v6.x and earlier
- **New Class**: The new class name/path to use in v7.0.0+
- **Full Namespace**: Complete namespace path including class name
- **Type**: `class`, `interface`, or `trait`
- **Category**: Functional grouping
- **Priority**: Migration urgency (HIGH/MEDIUM/LOW)

## Core Classes

### Exceptions

| Old Class | New Class | Full Namespace | Type | Priority |
|-----------|-----------|----------------|------|----------|
| `Application_Exception` | `ApplicationException` | `Application\Exception\ApplicationException` | class | HIGH |
| `Application_Exception_DisposableDisposed` | `DisposableDisposedException` | `Application\Disposables\DisposableDisposedException` | class | MEDIUM |
| `Application_Exception_UnexpectedInstanceType` | N/A (removed) | Use `\AppUtils\ClassHelper` methods | class | MEDIUM |

**Notes**: 
- `Application_Exception` is the most commonly used exception class
- Use `throw new \Application\Exception\ApplicationException()` instead
- The `UnexpectedInstanceType` exception has been removed; use `ClassHelper::requireInstanceOf()` instead

### Interfaces

| Old Class | New Class | Full Namespace | Type | Priority |
|-----------|-----------|----------------|------|----------|
| `Application_Interfaces_Admin_CollectionCreate` | `RecordCreateScreenInterface` | `DBHelper\Admin\Traits\RecordCreateScreenInterface` | interface | HIGH |
| `CollectionSettingsExtendedInterface` | `RecordEditScreenInterface` | `DBHelper\Admin\Traits\RecordEditScreenInterface` | interface | HIGH |
| `Application_Admin_ScreenInterface` | `AdminScreenInterface` | `Application\Interfaces\Admin\AdminScreenInterface` | interface | HIGH |

**Notes**:
- These interfaces are frequently implemented by admin screens
- The new interfaces are in the `DBHelper` package or `Application\Interfaces` namespace

### Area Classes

| Old Class | New Class | Full Namespace | Type | Priority |

- All area classes should now extend `BaseArea`

## Migration Examples

### Exception Usage

**Before (v6.x)**:
```php
throw new Application_Exception('Error message', 'Developer info');
```

**After (v7.0.0)**:
```php
use Application\Exception\ApplicationException;

throw new ApplicationException('Error message', 'Developer info');
```

### Admin Screen Interfaces

**Before (v6.x)**:
```php
class MyCreateScreen extends Application_Admin_Area_Mode_Submode_CollectionCreate
{
    // ...
}
```

**After (v7.0.0)**:
```php
use Application\Admin\Area\Mode\BaseRecordCreateSubmode;

class MyCreateScreen extends BaseRecordCreateSubmode
{
    // ...
}
```

### Interface Implementation

**Before (v6.x)**:
```php
class MyScreen implements Application_Interfaces_Admin_CollectionCreate
{
    // ...
}
```

**After (v7.0.0)**:
```php
use DBHelper\Admin\Traits\RecordCreateScreenInterface;

class MyScreen implements RecordCreateScreenInterface
{
    // ...
}
```

## Priority Definitions

- **HIGH**: Frequently used classes that appear in most applications. Migrate immediately.
- **MEDIUM**: Moderately used classes. Migrate during normal development cycles.
- **LOW**: Rarely used classes or edge cases. Migrate when encountered.

## Backward Compatibility

All deprecated classes in `src/classes/_deprecated/` extend or implement their new counterparts, providing transparent backward compatibility. However:

- **v7.0.0**: Deprecated classes available with `@deprecated` warnings
- **v7.1.0**: Deprecated classes still available (Q2 2026)
- **v8.0.0**: **Deprecated classes will be REMOVED** (Q3 2026)

**⚠️ Recommendation**: Migrate immediately to avoid breaking changes in v8.0.0.

## Search and Replace Patterns

For bulk migration, use these search/replace patterns (with care):

### Using `extends`

| Search Pattern | Replace With |
|----------------|--------------|
| `extends Application_Admin_Area` | `extends BaseArea` |
| `extends Application_Admin_Area_Mode` | `extends BaseMode` |
| `extends Application_Admin_Area_Mode_Submode` | `extends BaseSubmode` |
| `extends Application_Exception` | `extends ApplicationException` |

### Using `new`

| Search Pattern | Replace With |
|----------------|--------------|
| `new Application_Exception(` | `new ApplicationException(` |
| `new Application_Exception_DisposableDisposed(` | `new DisposableDisposedException(` |

**⚠️ Warning**: Always review search/replace results carefully. Add appropriate `use` statements to each file.
