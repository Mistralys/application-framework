### Implementation Summary
- Made `addWhere(string $statement): self` public on `BaseLookupItem` (was `protected`). Backward-compatible — existing subclass callers are unaffected.
- Added `private int $limit = 0` property and `public function setLimit(int $limit): self` fluent setter on `BaseLookupItem`.
- Added `public function findMatchingIDs(array $terms): int[]` on `BaseLookupItem`. Performs the same ID-collection logic as `findMatches()` but skips the UI rendering step (`renderLabel()`, `getURL()`, `addResult()`). Enforces the global limit cap via `array_slice()` after deduplication.
- Modified `findMatchesBySearch()` (private) to append `LIMIT $this->limit` to the SQL query when a limit is set (per-query performance optimization).
- Refactored `findMatches()` to delegate to `findMatchingIDs()` internally, then iterate IDs to hydrate results. This eliminates logic duplication between the two entry points.
- Implicit bug fix in `findMatches()`: the original code used `$ids = array($name)` for numeric terms (which overwrote the accumulated ID list instead of appending). The refactoring through `findMatchingIDs()` corrects this to `$ids[] = (int)$name`. This is a net improvement; the original behavior was only correct by coincidence for single-numeric-term lookups, which is the only real use case.
